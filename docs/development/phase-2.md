### 统一配置与跨窗口同步

配置升级为 schema v2。`normalizeAppSettings` 负责：

- 从阶段 1 快照补齐新字段并保留有效旧值
- 校验枚举、布尔值、时间、真实日历日期和允许的帧率
- 钳制并按步长归一化透明度与灵敏度
- 校验成对的物理像素坐标并保留负坐标
- 丢弃未知字段

schema v2 同时为后续同步模式和生活模式预留了第一版要求的配置字段，但本阶段只
建立数据契约、默认值和校验，不执行输入监听或活动调度。

设置写入从两个前端 WebView 各自的队列移到 Rust 进程级服务：

- `SettingsState` 使用 `Mutex` 串行化读取、patch 和 replace
- `AtomicU64` 为每次成功写入生成单调递增 revision
- 只有 `settings.json` 显式保存成功后才广播 `app-settings://changed`
- 保存失败时恢复内存中的旧值，不广播失败快照
- 两个 WebView 订阅同一设置事件
- 前端拒绝 revision 小于当前值的迟到响应
- 前端规范化修复先做递归语义比较：对象键顺序不影响结果，数组顺序仍有意义
- 只有原始值与规范化值语义不同时才发起修复，避免仅因 JSON 键顺序产生重复写入
- 修复使用带 `expectedRevision` 的条件 replace；Rust 在同一把锁内检查 revision，
  形成 compare-and-swap（CAS），避免旧修复覆盖并发新值
- CAS 冲突时前端重新读取最新快照并重试，最多执行三次修复重试

普通字段修改仍使用锁内原子 patch；用户明确执行“恢复全部默认设置”时使用无
`expectedRevision` 的完整 replace。CAS 的职责仅是保护自动规范化修复，不代表
所有设置写入都采用条件替换。

Rust 负责原子化读改写和磁盘提交，TypeScript 负责字段级规范化。前端不再直接持有
Tauri Store 权限。

### Windows 与桌宠集成

开机启动使用官方 `tauri-plugin-autostart`。设置窗口首次载入时读取操作系统实际
状态并修正配置。切换命令会记录原系统状态、执行启用或禁用、再次读取并确认结果，
然后才写入统一配置；验证失败或配置落盘失败时，会尝试补偿恢复先前的系统状态。恢复
默认设置会先关闭开机启动，再重置配置并居中桌宠；如果在配置重置完成前失败，设置
窗口还会补偿恢复先前的开机启动状态。

桌宠窗口新增原生权限用于调整窗口尺寸和置顶。桌宠收到设置快照后会即时应用：

- 原生窗口尺寸
- CSS 透明度
- 始终置顶
- 位置锁定
- 记住位置行为

桌宠右键菜单增加尺寸子菜单和“始终置顶”复选项，并继续保留打开设置、锁定位置和
退出应用。

桌宠初始化时先应用尺寸和置顶；只有启用位置记忆且两项坐标都有效时才恢复物理
坐标，否则在新尺寸生效后执行不落盘的居中。用户主动恢复默认位置时，Rust 会先
记录原点，居中后把实际坐标写入配置；若配置保存失败，则尝试把原生窗口移回原点。

原生窗口移动事件使用 `220ms` 防抖计时器保存最后坐标。每次需要记忆的移动事件
都会推进 generation；关闭“记住位置”、收到外部坐标更新或组件卸载时会取消计时器
并使旧 generation 失效，避免迟到计时器把新状态覆盖回旧坐标。重新开启“记住位置”
时会立即读取当前原生窗口外层坐标并写入配置。尺寸和置顶更新则进入串行 appearance
队列，避免快速快照交错执行原生窗口操作。

## 歧义处理

### 七页骨架与分阶段开发

“创建全部页面骨架”解释为七项导航和七个可访问页面都存在；只有常规、桌宠和性能
页进入本阶段要求的设置闭环。同步模式、生活模式和角色页不提前实现后续业务。

### 完整配置字段与后续功能

总体需求列出了第一版必须持久化的同步模式、生活模式和性能字段。为避免后续多次
破坏性迁移，schema v2 一次建立这些字段的类型、默认值和校验，但字段存在不代表
对应运行系统已经实现。

### 启动时显示桌宠

当前没有系统托盘。如果允许关闭“启动时显示桌宠”，应用可能在首次启动后完全不可
见。因此本阶段保留该字段，但界面固定为开启；等备用入口形成闭环后再开放。

### 关闭设置窗口后的行为

阶段 1 已确定关闭设置窗口只隐藏窗口、桌宠继续运行。本阶段保留
`continueRunningOnSettingsClose` 字段，但界面和 Rust 生命周期继续固定执行隐藏，
避免破坏既有验收条件。

### 默认启动模式

除同步和生活模式外，增加“沿用上次模式”，用于表达总体需求中的“重启恢复上次
模式”。当前仅保存该偏好，实际启动模式选择将在模式系统接入时实现。

### 性能设置

性能页本阶段负责合法值、即时保存和重启保留，不宣称已经限制帧率或检测全屏程序。
所有运行时效果明确延期到阶段 8。
